1 Introduction

Weâ€™ll illustrate the basic components of a Stan program with a simple linear regression - the model is easy to understand so understanding the model doesnâ€™t get in the way of learning basics of Stan code.

Iâ€™ll start out as simply as possible and then add tips and elements that can be carried over to other models.

Note

Much of what is on this page isnâ€™t necessarily a best practice. In fact, I wouldnâ€™t write this model in raw Stan at all. Iâ€™d use rstanarm or brms - they both provide a ton of functionality that takes unnecessary extra effort to do in raw Stan. But Iâ€™ll write this in Stan in such a way as to highlight certain aspects of a Stan program and give tips that can be carried over to other models.

2 Data

First weâ€™ll create some data of the form \[ y_i = \alpha + \beta x_i + \epsilon_i, \;\;\;\;\;i = 1, \ldots, n, \]

and weâ€™ll use true values \(\alpha = 2\), \(\beta = 3\), and \(\sigma = 6\).

3 Model 1 - As Simple as Possible

First, we will write out the model as simply as possible with no frills. Donâ€™t write it this way in practice - use some of the tips shown later on this page.

3.1 Statistical Model

Weâ€™ll write down the full statistical model:

\[ \begin{align} Y_i \; | \; \alpha, \beta, \sigma &\sim Normal(\alpha + \beta x_i, \; \sigma) \\ \alpha &\sim Normal(0, 30) \\ \beta &\sim Normal(0, 10) \\ \sigma &\sim Half-Normal(0, 15) \\ \end{align} \]

Note

The above normal distributions are parameterized with the standard deviation rather than the variance to remain consistent with the parameterization that Stan and R use.

3.2 Stan Model

data{
  
  int<lower = 1> n;
  array[n] real y;
  vector[n] x;
  
}
parameters{
  
  real alpha;
  real beta;
  real<lower = 0> sigma;

}
model{
  
  // Priors
  alpha ~ normal(0, 30);
  beta ~ normal(0, 10);
  sigma ~ normal(0, 15);
  
  // Likelihood
  y ~ normal(alpha + beta*x, sigma);

}

As can be seen in the model, the data block is where we input the data, the parameters block is where we declare the parameters, and the model block is where we write the model.

Important

Make sure to declare any constraints in the parameter block. Hence, the lower bound on \(\sigma\) - real<lower = 0> sigma;.

4 Model 2 - Donâ€™t Hardcode Things If You Might Want to Adjust Them

Youâ€™ll notice in the previous model that we hardcoded the hyperparameters for the priors on \(\alpha, \; \beta,\) and \(\sigma\). Thatâ€™s generally not great practice - every time you want to change one of these hyperparameters, you have to recompile the model. Instead, declare these hyperparameters in the data block and input them with the observed data.

Tip

Itâ€™s easier for your workflow to hardcode only things that you willl never change for any reason.

4.1 Statistical Model

Weâ€™ll write down the full statistical model:

\[ \begin{align} Y_i \; | \; \alpha, \beta, \sigma &\sim Normal(\alpha + \beta x_i, \; \sigma) \\ \alpha &\sim Normal(\mu_\alpha, sd_\alpha) \\ \beta &\sim Normal(\mu_\beta, sd_\beta) \\ \sigma &\sim Half-Normal(\mu_\sigma, sd_\sigma) \\ \end{align} \]

4.2 Stan Model

data{
  
  int<lower = 1> n;
  array[n] real y;
  vector[n] x;
  
  real mu_alpha, mu_beta, mu_sigma;
  real<lower = 0> sd_alpha, sd_beta, sd_sigma;
  
}
parameters{
  
  real alpha;
  real beta;
  real<lower = 0> sigma;

}
model{
  
  // Priors
  alpha ~ normal(mu_alpha, sd_alpha);
  beta ~ normal(mu_beta, sd_beta);
  sigma ~ normal(mu_sigma, sd_sigma);
  
  // Likelihood
  y ~ normal(alpha + beta*x, sigma);

}

5 Model 3 - Keep the Mean Prediction (\(\mu_i = \alpha + \beta x_i\))

Youâ€™ll notice in the posterior summaries above (provided by the $summary() call) that we are getting posterior summaries for lp__, alpha, beta, and sigma. We might want to keep the linear prediction, \(\mu_i = \alpha + \beta x_i\). The best way to do this is to put it in transformed parameters.

5.1 Statistical Model

Weâ€™ll write down the full statistical model3:

\[ \begin{align} Y_i \; | \; \alpha, \beta, \sigma &\sim Normal(\mu_i, \; \sigma) \\ \alpha &\sim Normal(\mu_\alpha, sd_\alpha) \\ \beta &\sim Normal(\mu_\beta, sd_\beta) \\ \sigma &\sim Half-Normal(\mu_\sigma, sd_\sigma) \\ \mu_i &= \alpha + \beta x_i \end{align} \]

5.2 Stan Model

data{
  
  int<lower = 1> n;
  array[n] real y;
  vector[n] x;
  
  real mu_alpha, mu_beta, mu_sigma;
  real<lower = 0> sd_alpha, sd_beta, sd_sigma;
  
}
parameters{
  
  real alpha;
  real beta;
  real<lower = 0> sigma;

}
transformed parameters{

  vector[n] mu = alpha + beta*x;

}
model{
  
  // Priors
  alpha ~ normal(mu_alpha, sd_alpha);
  beta ~ normal(mu_beta, sd_beta);
  sigma ~ normal(mu_sigma, sd_sigma);
  
  // Likelihood
  y ~ normal(mu, sigma);

}

Tip

In general, if thereâ€™s a quantity that you both want to keep and is necessary to have in the model block, e.g. a parameter in the likelihood like mu here or like an individual prediction (IPRED) in a PopPK model, put it in transformed parameters. If itâ€™s something that you want to keep, but isnâ€™t necessary for the model block, itâ€™s better to go in generated quantities - the evaluations in transformed parameters are done at every leapfrog step while the evaluations in generated quantities are only performed once at the end of each sampling iteration. No need to make a calculation at every leapfrog step if it isnâ€™t needed to increment the target4.

6 Model 4 - Write a Function

It is sometimes easier and cleaner to write your own functions to do some calculations in your Stan program. Here, weâ€™ll add a functions block and write a function to calculate mu and use that function in transformed parameters where mu is calculated.

6.1 Stan Model

functions{

  vector calculate_mu(real intercept, real slope, vector x){

    vector[num_elements(x)] mu = intercept + slope*x;
    return mu;

  }

}
data{

  int<lower = 1> n;
  array[n] real y;
  vector[n] x;

  real mu_alpha, mu_beta, mu_sigma;
  real<lower = 0> sd_alpha, sd_beta, sd_sigma;

}
parameters{

  real alpha;
  real beta;
  real<lower = 0> sigma;

}
transformed parameters{

  vector[n] mu = calculate_mu(alpha, beta, x);

}
model{

  // Priors
  alpha ~ normal(mu_alpha, sd_alpha);
  beta ~ normal(mu_beta, sd_beta);
  sigma ~ normal(mu_sigma, sd_sigma);

  // Likelihood
  y ~ normal(mu, sigma);

}

7 Model 5 - Calculate Other Things You Might Want (e.g. posterior predictive checks, predictions, log-likelihoods, â€¦)

Now I want to make predictions at a new set of x-values. Iâ€™ll also do some posterior predictive checks (PPCs) and calculate log-likelihoods of individual observations. Since I donâ€™t need any of these to evaluate the posterior distribution, I do these things in the generated quantities block.

7.1 Stan Model

functions{

  vector calculate_mu(real intercept, real slope, vector x){

    vector[num_elements(x)] mu = intercept + slope*x;
    return mu;

  }

}
data{

  int<lower = 1> n;
  array[n] real y;
  vector[n] x;

  real mu_alpha, mu_beta, mu_sigma;
  real<lower = 0> sd_alpha, sd_beta, sd_sigma;

  int<lower = 1> n_pred;
  vector[n_pred] x_pred;

}
parameters{

  real alpha;
  real beta;
  real<lower = 0> sigma;

}
transformed parameters{

  vector[n] mu = calculate_mu(alpha, beta, x);

}
model{

  // Priors
  alpha ~ normal(mu_alpha, sd_alpha);
  beta ~ normal(mu_beta, sd_beta);
  sigma ~ normal(mu_sigma, sd_sigma);

  // Likelihood
  y ~ normal(mu, sigma);

}
generated quantities{

  // PPC - mu is still accessible here, so it doesn't need to be calculated again
  array[n] real y_ppc = normal_rng(mu, sigma);
  
  // Predictions at new points
  vector[n_pred] y_pred_mean = calculate_mu(alpha, beta, x_pred);
  array[n_pred] real y_pred = normal_rng(y_pred_mean, sigma);
  
  // Log-likelihoods of individual observations
  vector[n] log_lik;
  for(i in 1:n){
    log_lik[i] = normal_lpdf(y[i] | mu[i], sigma);
  }

}

8 Model 6 - Modify the Data in transformed data

Up to here, weâ€™ve touched on all program blocks except for transformed data. This is where we can declare and define new variables (if Iâ€™m going to hardcode something, it will be in this block) or modify variables from data to use in the rest of the program.

For this example, we will center our independent variable (by subtracting off the mean) to give the intercept a more meaningful interpretation.

8.1 Stan Model

functions{

  vector calculate_mu(real intercept, real slope, vector x){

    vector[num_elements(x)] mu = intercept + slope*x;
    return mu;

  }

}
data{

  int<lower = 1> n;
  array[n] real y;
  vector[n] x;

  real mu_alpha, mu_beta, mu_sigma;
  real<lower = 0> sd_alpha, sd_beta, sd_sigma;

  int<lower = 1> n_pred;
  vector[n_pred] x_pred;

}
transformed data{

  real mean_x = mean(x);
  vector[n] x_centered = x - mean(x);
  vector[n_pred] x_pred_centered = x_pred - mean(x);

}
parameters{

  real alpha;
  real beta;
  real<lower = 0> sigma;

}
transformed parameters{

  vector[n] mu = calculate_mu(alpha, beta, x_centered);

}
model{

  // Priors
  alpha ~ normal(mu_alpha, sd_alpha);
  beta ~ normal(mu_beta, sd_beta);
  sigma ~ normal(mu_sigma, sd_sigma);

  // Likelihood
  y ~ normal(mu, sigma);

}
generated quantities{

  // PPC - mu is still accessible here, so it doesn't need to be calculated again
  array[n] real y_ppc = normal_rng(mu, sigma);
  
  // Predictions at new points
  vector[n_pred] y_pred_mean = calculate_mu(alpha, beta, x_pred_centered);
  array[n_pred] real y_pred = normal_rng(y_pred_mean, sigma);
  
  // Log-likelihoods of individual observations
  vector[n] log_lik;
  for(i in 1:n){
    log_lik[i] = normal_lpdf(y[i] | mu[i], sigma);
  }

}

9 Model 7 - Use Local Variables for Interim Quantities that You Donâ€™t Want to Keep

Building off of Model 5 in Section 7.1, letâ€™s imagine a case where we didnâ€™t want to keep y_pred_mean, but we still wanted to have it to help create y_pred. In this situation, we declare y_pred, but donâ€™t assign it a value, and then create a statement block with curly brackets. In that statement block, we can declare, assign, and perform calculations with interim values that we donâ€™t want to keep, such as y_pred_mean. An assignment made to a variable declared outside of the statement block will be saved, even if the assignment happens inside the statement block. This model shows how this is done:

9.1 Stan Model

functions{

  vector calculate_mu(real intercept, real slope, vector x){

    vector[num_elements(x)] mu = intercept + slope*x;
    return mu;

  }

}
data{

  int<lower = 1> n;
  array[n] real y;
  vector[n] x;

  real mu_alpha, mu_beta, mu_sigma;
  real<lower = 0> sd_alpha, sd_beta, sd_sigma;

  int<lower = 1> n_pred;
  vector[n_pred] x_pred;

}
parameters{

  real alpha;
  real beta;
  real<lower = 0> sigma;

}
transformed parameters{

  vector[n] mu = calculate_mu(alpha, beta, x);

}
model{

  // Priors
  alpha ~ normal(mu_alpha, sd_alpha);
  beta ~ normal(mu_beta, sd_beta);
  sigma ~ normal(mu_sigma, sd_sigma);

  // Likelihood
  y ~ normal(mu, sigma);

}
generated quantities{

  // PPC - mu is still accessible here, so it doesn't need to be calculated again
  array[n] real y_ppc = normal_rng(mu, sigma);
  
  // Predictions at new points
  array[n_pred] real y_pred;  // included in output
  
  // Log-likelihoods of individual observations
  vector[n] log_lik;
  for(i in 1:n){
    log_lik[i] = normal_lpdf(y[i] | mu[i], sigma);
  }
  
  // Start of statement block
  {
  
    vector[n_pred] y_pred_mean = calculate_mu(alpha, beta, x_pred); // not included in output
    y_pred = normal_rng(y_pred_mean, sigma);
  
  }
  
}

10 Summary

Hopefully this shows a little about the program blocks in a Stan program. Two takeaways:

  1. Donâ€™t hardcode anything you might want to change or tinker with - itâ€™ll help your model-writing to not have to recompile the model over and over.
  2. Youâ€™ll likely calculate certain quantities that arenâ€™t explicitly in the model. We want to think carefully about which block is the most computationally efficient place to calculate those quantities - this particular example is extremely easy, so computational cost isnâ€™t a concern, but most models in PK/PD are more computationally expensive:
    1. If you want to keep it for posterior analysis and it is involved in incrementing the target, put it in transformed parameters (like mu in our examples).
    2. If you want to keep it for posterior analysis and it is not involved in incrementing the target, put it in generated quantities (like y_pred in our examples).
    3. If you donâ€™t want to keep it for posterior analysis and it is involved in incrementing the target, do the calculations in the model block (I donâ€™t have one like this in these examples, since I rarely, if ever, do this)
    4. If you donâ€™t want to keep it for posterior analysis and it is not involved in incrementing the target, donâ€™t do the calculations at all.
